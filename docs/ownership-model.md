# Ownership Model

The current boundary is intentionally sharper than the old framework boundary.

Trellis owns the reliable GTM agent stack. Builders own the business truth.

## Trellis Owns

- deployment wiring
- agent harness integration
- runtime routes
- durable state schema
- knowledge and skill pack loading
- signal normalization
- prospect, draft, approval, workflow, and audit records
- no-send and approval gates
- MCP and dashboard surfaces
- provider connection manifests
- doctor, smoke, and deploy checks

## Builders Own

- product facts
- ICP definitions
- qualification judgment
- copy style
- compliance constraints
- handoff rules
- provider credentials
- business-specific workflow branches

## Coding Agents Own

Coding agents should help edit the builder-owned layer:

- add markdown knowledge
- write or revise skills
- connect providers
- inspect doctor failures
- add narrow custom workflow steps

They should not be required to assemble the runtime substrate.

## Rule

If a team needs it before their first credible GTM agent demo, Trellis should probably own it.

If it changes because the company, ICP, product, or sales motion changed, the builder should own it.
